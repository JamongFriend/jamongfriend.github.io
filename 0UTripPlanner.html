0U Trip Planner

기술 스택: Node.js, Express, MySQL, React.js, Kakao API

프로젝트 개요

0U Trip Planner는 여행 일정을 작성하고 공유할 수 있는 웹 플랫폼입니다. 백엔드는 Node.js와 Express로 RESTful API를 설계하였으며, 프론트엔드는 React와 Android Studio 환경에서 동일한 API를 활용해 구현했습니다. 사용자 인증, 여행 일정 CRUD, 지도 기반 장소 검색 등 실제 서비스 흐름을 고려한 구조로 개발했습니다.

주요 기능

  • 회원가입 및 로그인: /api/auth/login, /api/auth/signup 쿠키 + 세션 기반 인증, 비밀번호 해싱 처리(bcrypt)
  • 여행 계획 CRUD: /api/trips 여행 일정 생성, 조회, 수정, 삭제 기능 (비동기 처리 async/await 기반)
  • 지도 기반 장소 검색: Kakao Map API 연동 → 여행지 좌표 및 장소 정보 실시간 표시
  • 소셜 로그인: Kakao Login API으로 간편 로그인 및 사용자 프로필 연동
  • 클라이언트 연동: React 및 Android 앱 모두 동일 REST API를 호출하여 데이터 일관성 확보
  • CORS 보안 처리: Express 미들웨어에서 cors() 설정으로 교차 출처 요청 허용
  • 데이터베이스 설계: MySQL 기반 User, Trip, Plan, Place 테이블 구조화 및 관계 설정 (1:N, N:M)

아키텍처 & 설계 포인트

  • 3계층 구조: Router → Service → Repository 계층 분리로 모듈화 및 유지보수성 확보
  • RESTful 설계 원칙: HTTP 메서드별(POST/GET/PUT/DELETE) 명확한 역할 정의
  • 세션 관리: express-session과 cookie-parser를 이용한 세션 유지 및 만료 처리
  • 비동기 제어: async/await + try/catch 기반 오류 처리로 비동기 흐름 안정화
  • 공통 미들웨어 관리: 인증, 로깅, CORS, 에러 핸들링을 app.use() 레벨에서 중앙 관리
  • 확장성 고려: 향후 일정 공유 기능 추가 시에도 기존 API 구조 수정 없이 확장 가능하도록 설계

주요 시나리오

  • 회원가입 → 로그인: 사용자는 이메일/비밀번호 입력 → 세션 생성 후 메인 페이지 이동
  • 여행 일정 등록: 사용자가 여행 이름, 날짜, 장소 입력 → MySQL Trip 테이블 저장
  • 지도 검색: Kakao Map API를 통해 장소 검색 후 좌표 기반 마커 표시
  • 계획 수정/삭제: 본인 소유 일정만 접근 가능하도록 세션 사용자 ID 검증
  • Android 연동: 동일 REST API를 모바일 클라이언트에서 호출하여 웹과 데이터 공유

트러블슈팅 & 문제 해결

  • 비동기 제어 혼선: Promise 체인 중첩 → async/await 리팩토링으로 흐름 단순화
  • CORS 정책 충돌: 프론트·모바일 연동 시 교차 출처 에러 → CORS 미들웨어 설정으로 해결
  • 세션 만료 이슈: cookie-parser와 express-session 설정 분리 → 세션 지속성 정상화
  • 데이터 무결성: 외래키 제약조건 적용 → 일정 삭제 시 연관 데이터 자동 삭제
