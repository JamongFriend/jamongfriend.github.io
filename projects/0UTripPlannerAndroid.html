0U Trip Planner (Android Ver.)

⚙️ 기술 스택: Java, Android Studio, Retrofit, Node.js(Express), REST API

프로젝트 개요

0U Trip Planner(Android)는 기존 웹 프로젝트의 기능을 모바일 환경에서 사용할 수 있도록 확장한 버전입니다. 웹 버전과 동일한 Node.js 기반 REST API를 활용하여 여행 계획 생성·수정·조회 등 핵심 기능을 그대로 제공하며, 모바일 UI/UX에 맞춘 화면 구성과 HTTP 통신 구조를 추가 구현하였습니다.

주요 기능

  • 회원가입 및 로그인: 이메일·비밀번호 기반 인증 → 세션 또는 토큰을 통한 지속적 로그인 유지
  • 여행 계획 조회·상세 화면: 백엔드 Trip API 연동 → 사용자 일정 목록/상세/필터링 제공
  • 여행 계획 CRUD: 생성, 수정, 삭제 API 호출을 통한 실시간 일정 관리
  • HTTP 통신: Retrofit 기반 REST API 요청/응답 처리
  • React 웹과 데이터 일관성 유지: 웹/모바일 모두 동일 백엔드 엔드포인트 사용

아키텍처 & 설계 포인트

  • MVC 기반 화면 구성: Activity/Fragment → ViewModel → Repository 구조 적용
  • HTTP 클라이언트 구조화: Retrofit + Gson으로 데이터 직렬화 및 비동기 처리 안정화
  • 세션/로그인 유지 처리: SharedPreferences 기반 사용자 정보 캐싱
  • 비동기 통신 안정화: try/catch + 콜백 구조 정리 → 네트워크 에러 핸들링 강화
  • 재사용 가능한 API Service 모듈: Base URL · Header · OkHttp Interceptor 구성

주요 시나리오

  • 회원가입/로그인: 사용자가 이메일·비밀번호 입력 → 서버 인증 → 사용자 Session/Token 저장
  • 여행 계획 조회: 앱 실행 시 사용자 ID로 Trip 목록 불러오기 → RecyclerView로 UI 구성
  • 여행 계획 상세 보기: 선택한 Trip ID로 상세 정보 API 호출 → 화면 전환
  • 여행 계획 수정/삭제: API 요청 후 성공 여부에 따라 UI 즉시 갱신

트러블슈팅 & 문제 해결

  • CORS / 네트워크 차단 이슈: 모바일 WebView 요구사항 불일치 → 백엔드에서 CORS 설정 강화로 해결
  • Retrofit 네트워크 실패: 비동기 콜백 누락 문제 발견 → 공통 Response Wrapper 적용해 안정화
  • 로그인 유지 장애: SharedPreferences 저장 시점 누락 → 로그인 성공 즉시 값 저장하도록 구조 변경
  • 화면 전환 시 데이터 전달 문제: Intent/Bundle 구조 정돈 → Trip ID 기반 서버 요청방식으로 통일
