Library Management System

기술 스택: Java, Spring Boot, JPA, MySQL

프로젝트 개요

도서 등록/대여/반납/예약 전 과정을 관리하는 백엔드 시스템입니다. 도메인–리포지토리–서비스–컨트롤러의 계층 구조로 설계했고, Admin*/App* 컨트롤러를 분리해 관리자 화면과 사용자 기능을 명확히 구분했습니다. 국립중앙도서관 OpenAPI와 연동하여 ISBN으로 책 정보를 조회·자동 등록하는 기능을 제공합니다.

주요 기능

  • ISBN 조회 & 자동 등록/증가: /api/v1/admin/books/lookup-isbn로 외부 OpenAPI 조회 → 기존 재고면 증가, 없으면 신규 생성 (RestTemplate + ObjectMapper)
  • 대여/반납: 대여 생성, 반납 처리 시 재고 복원 및 다음 예약자 자동 배정
  • 예약 우선 처리: 같은 도서에 대해 가장 빠른 예약자(earliest by reservationDate)에게 자동 대여 할당
  • 대여 기간 자동 감소: 매일 00:00(Asia/Seoul) 스케줄러로 남은 대여일 수 감소
  • 중복 대여 방지: 회원·도서 기준 활성 대여 존재 여부 검사 후 차단
  • 재고 동시성 제어: 재고 변경 시 PESSIMISTIC_WRITE 사용으로 동시 수정 안전성 확보
  • 보안: Spring Security 기반 커스텀 로그인/로그아웃, 비밀번호 BCrypt 해시 적용

아키텍처 & 설계 포인트

  • 계층형 아키텍처: Domain / Repository / Service / Controller
  • 관리자/사용자 컨트롤러 분리: AdminBookController, AdminReservationController / AppBookController, AppReservationController
  • DTO 응답 최적화: 엔티티 직접 노출을 지양하고 API 전용 DTO 사용
  • 쿼리 최적화: 필요 시 페치 전략 조정, 예약·대여 조회 전용 쿼리 분리

주요 시나리오

① ISBN으로 등록

입력된 ISBN이 이미 있으면 재고만 +N, 없으면 API 조회 후 생성

POST /api/v1/admin/books

② 대여 생성

회원·도서 유효성 체크 → 중복 대여 방지 → 재고 감소 → 대여 레코드 생성

POST /rentals/new

③ 반납 & 자동 배정

반납 처리 후 재고 복원 → 동일 도서의 가장 이른 예약자에게 자동 대여

POST /rentals/return

④ 자정 스케줄러

매일 00:00에 활성 대여의 남은 일수 감소(@Scheduled)

CRON: 0 0 0 * * *

트러블슈팅 & 문제 해결

  • 중복 대여 생성
    원인: 동일 회원/도서에 활성 대여 존재 여부 체크 누락
    해결: RentalRepository.existsActiveByMemberAndBook 선검증, 서비스 레벨 가드 추가
    성과: 동일 도서 중복 대여 100% 차단
  • ISBN 입력 포맷 다양(‘-’ 포함/미포함)으로 조회 실패
    원인: 외부 API/내부 DB에서 포맷 불일치
    해결: normalizeIsbn()로 하이픈 제거·길이 검증, 10→13 변환 처리, 유효성 예외 공통화
    성과: ISBN 조회 성공률 향상, 운영 입력 실수 감소
